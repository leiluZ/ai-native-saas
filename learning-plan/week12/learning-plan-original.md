## Day 1：核心技术复盘（FastAPI/Agent/RAG/vLLM）

- **目标**：建立核心知识卡片，掌握底层原理、边界条件与常见追问，形成 2 分钟精准表达能力。
- **实操**：
  1. 绘制技术脑图：FastAPI 异步事件循环/依赖注入、Agent 状态机与 Tool Calling、RAG 检索链路（分块/Embedding/重排/去重）、vLLM 推理管线（PagedAttention/KV Cache/Chunked Prefill）。
  2. 针对每个模块准备「原理 → 优势 → 瓶颈 → 替代方案 → 项目实践」五段式话术。
  3. 使用闪卡工具（Anki/Notion）进行间隔重复，配合计时器控制单点输出 ≤ 90s。

## Day 2：系统设计 #1（0-1 SaaS 架构/多租户/成本）

- **目标**：掌握从零构建 AI SaaS 的标准架构范式，重点攻克数据隔离、路由分发与成本核算。
- **实操**：
  1. 明确需求约束：租户规模、SLA、数据合规（PIPL/GDPR）、成本上限、扩展节奏。
  2. 设计三层架构：接入层（Nginx/API Gateway/RateLimit）→ 业务层（FastAPI 微服务/Celery 异步）→ 推理/数据层（vLLM 集群/向量库/多租户 DB 隔离策略）。
  3. 对比多租户方案：独立 DB vs 共享 Schema vs 行级隔离（tenant_id），输出选型矩阵与迁移路径。
  4. 成本控制：Token 分级路由（GPT-4 vs GPT-4o-mini vs Local Model）、缓存策略、单用户月度配额 $50。
  5. 白板 10 分钟绘制架构图，标注数据流、关键组件、SLO 监控点。
- **Prompt 模板**：

```md

1. 给出标准化 Prompt：多租户 AI 写作平台，日活 1w，需支持按租户计费、数据隔离、模型路由
2. 输出架构设计模板：需求澄清 → 估算 → 高层架构 → 核心模块 → 权衡取舍 → 演进路线
3. 提供多租户隔离对比表（安全/成本/运维/扩展性）与适用场景
4. 附 10 分钟白板绘图 Checklist：组件/数据流/监控点/降级路径
5. 生成 3 个 Follow-up：如某租户突发流量暴涨、账单对账不一致、合规审计要求数据物理隔离
```

- **验收**：
  - ✅ 10 分钟内白板画出完整架构，标注清晰无遗漏
  - ✅ 多租户隔离方案选型合理，附数据支撑

## Day 3：系统设计 #2（高并发 AI 服务/容灾/降级）

- **目标**：设计高可用、高并发生成式 AI 服务，掌握容灾预案、降级链路与故障隔离机制。
- **实操**：
  1. 定义 SLO：可用性 99.9%、P99 ≤ 2s、错误率 ≤ 1%，明确 Error Budget。
  2. 高并发方案：异步 I/O + 连接池 + vLLM Continuous Batching + KV Cache。
  3. 容灾设计：多可用区部署、主备 vLLM 实例、DNS 故障切换。
  4. 构建降级链路（4 层）：vLLM 本地 → Redis 缓存 → 轻量模型 → 云端 API Fallback → 优雅拒接。
  5. 雪崩防控：限流（滑动窗口）+ 熔断（Circuit Breaker）+ 隔离（Bulkhead）。
  6. 绘制高并发数据流图，标注熔断阈值、重试策略、背压队列与可观测性埋点。
- **Prompt 模板**：

```md
构建高并发 AI 服务架构设计演练模板，要求：

1. 约束条件：QPS 5000，长文本生成，GPU 资源受限，需保障 P99 < 2s
2. 架构输出：负载均衡 → 请求队列 → 推理网关（vLLM）→ 降级路由 → 监控告警
3. 容灾策略：多 AZ 部署、无状态扩容、权重/配置热更新、故障自动切换
4. 降级矩阵（4 层）：触发条件、降级目标、状态恢复条件、客户端感知控制
5. 雪崩防控：限流 100 req/s + 熔断 ErrorRate > 50% + 隔离（独立线程池）
6. 附 10 分钟白板 Checklist 与评分 Rubric（清晰度/完整性/权衡/可演进性）
```

- **验收**：
  - ✅ 架构图 10 分钟内完成，降级链路与容灾路径清晰
  - ✅ SLO 与 Error Budget 计算合理
  - ✅ 降级链路 4 层分级，雪崩防控 4 手段
  - ✅ 评分 Rubric 自查 ≥ 85/100

## 动态调整建议

- **临场表达紧张/卡顿**：
  - 采用 PREP 结构（Point-Reason-Example-Point）先抛结论再展开，降低忘词焦虑。
- **技术深度不足/答不出底层**：
  - 聚焦「边界条件」而非「背诵源码」，准备标准话术："我不确定细节，但我可以这样排查..."
- **项目经历与 JD 匹配度低**：
  - 强化「可迁移能力」包装：RAG 调优 → "检索系统性能优化"，vLLM 部署 → "高并发服务治理"。
- **时间紧迫/无法完整模拟**：
  - D5/D6 合并为"核心题型抽卡"模式：随机抽 3 道技术 + 1 道架构 + 1 道 Behavioral，限时 40min。
- **目标岗位偏算法/偏工程**：
  - 偏算法：D1 强化 Embedding/重排/评估指标；偏工程：D1 强化异步/连接池/监控/降级。
